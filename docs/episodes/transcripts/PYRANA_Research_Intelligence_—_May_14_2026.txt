HOST_B: Three agents in sequence. A Decompose Agent splits the question into sub-queries. HOST_A: So the program itself is the reasoning trace.

Fix the program, run it again.
HOST_A: So instead of asking the model "are you sure?" — which famously doesn't work — you let the Python interpreter tell it what's broken.
HOST_B: Right. That's the move. It's grounded error feedback, not LLM self-reflection. The second loop is execution-driven adaptive retrieval — if a sub-step returns "unknown" or thin evidence, it boosts the top-k for that specific call and tries again.
