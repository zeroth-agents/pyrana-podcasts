HOST_A: Okay so what's the actual mechanism. How does DeMem work.
HOST_B: They model long-horizon memory as a K-state runtime constraint. At answer time, the agent has K memory slots. A deterministic encoder maps each history-query pair into one of those K slots. HOST_A: And the key result is the forgetting boundary, Theorem 1.
HOST_B: Right. Two histories can safely share a memory slot if and only if there exists a common action that is epsilon-optimal for both. That's it. Purely decision-theoretic. Description doesn't enter.
HOST_A: Which is wild because it means you might cluster two semantically very different memories together if they imply the same action, and split two semantically identical memories apart if they imply different actions.
HOST_B: DeMem implements this online. Starts with one slot. Splits only when certified decision conflict is detected — two histories sharing a slot would require different optimal actions. Near-minimax regret guarantee.
