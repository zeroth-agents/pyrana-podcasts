Four papers land on the same axis: how to make agent reasoning auditable and reproducible when the underlying model is stochastic. The episode covers a multi-hop retrieval system that externalises reasoning as executable Python, a multi-agent patent framework that uses cross-methodology convergence to rank claims, a memory study showing that model-driven memory rewriting degrades accuracy over time, and a scientific-workflow paper making the empirical case for deterministic tool dispatch over generative code.

Three of the four papers map directly onto active Cortex work. The PyRAG executable-reasoning pattern is a working prototype for the Text2Cypher pathway: emit a Python program, log its execution trace as a CxU provenance event, and every multi-hop query becomes a replayable artifact. The memory paper is a concrete warning against shipping CORTEXLEARNINGENABLED on by default — repeated LLM rewrites corrupt previously accurate knowledge. And the deterministic-mediation paper is the strongest empirical backing yet for the inproc-MCP tool design: typed tools beat generated code on reproducibility and error traceability.

Papers covered:

  • Retrieval is Cheap, Show Me the Code: Executable Multi-Hop Reasoning for Retrieval-Augmented Generation — Turns multi-hop RAG reasoning into executable Python programs, yielding +25.5 exact-match on Bamboogle with no fine-tuning.
  • IdeaForge: A Knowledge Graph-Grounded Multi-Agent Framework for Cross-Methodology Innovation Analysis and Patent Claim Generation — Multiple agents using distinct innovation frameworks write to a shared knowledge graph; convergent claims are ranked higher for patent drafting.
  • Useful Memories Become Faulty When Continuously Updated by LLMs — Continuous LLM-driven memory rewriting degrades accuracy of previously correct knowledge over successive updates.
  • It's not the Language Model, it's the Tool: Deterministic Mediation for Scientific Workflows — Replacing generative code with typed, deterministic tool selection improves reproducibility and error traceability in scientific agent workflows.

Takeaways for our platform:

  • Wire the PyRAG Plan Agent pattern into the Text2Cypher pathway so multi-hop queries emit a Python execution trace logged as a CxU provenance event — this delivers the replayable audit trail the governed-agent story requires. Add a complexity router so single-hop lookups bypass program-synthesis overhead entirely.
  • Gate CORTEXLEARNINGENABLED to off-by-default and require explicit versioned checkpoints before any model-driven CxU rewrite is committed; the memory paper shows that incremental rewrites silently corrupt accurate claims without triggering obvious failure signals.
  • Use IdeaForge's convergence-tagging pattern as a reference architecture for the regulated-tenant demo: when multiple CxU extraction agents independently surface the same claim from different source types, promote that claim's confidence score and surface the agreement count in the audit trail.
